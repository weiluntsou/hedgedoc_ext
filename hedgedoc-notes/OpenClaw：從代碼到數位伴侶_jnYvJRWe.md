# OpenClaw：從代碼到數位伴侶




## 1. 核心哲學：零信任架構 (Zero Trust Architecture)

OpenClaw 將虛擬助理視為「身處異國的陌生人」，基於安全考量嚴格執行隔離與權限控制 。
 
**隔離環境 (Isolation)**：將助理限制在名為「The Tank」的隔離單元中 。
 
**權限控制 (Access Control)**：遵循最小權限原則，絕不授予 Root 權限或主電腦訪問權 。

**數據保護 (Data Safety)**：將加密貨幣私鑰與主要電子郵件進行物理隔離與加密 。



## 2. 基礎設施：骨架結構與您的實戰佈署

系統需要一個安全的環境來運行，以確保穩定與進程隔離。

**棲息地 (The Habitat)**：
**官方建議**：推薦使用 Hostinger KVM2 VPS，具備備份與災難恢復功能 。


* **實戰佈署**： OpenClaw 與相關 AI 服務架設在回收再利用的老舊硬體上，例如 **2011 年的 Mac Mini**、**Raspberry Pi**，甚至是除役的手機（**Xiaomi Max 2**、**Pixel 4**）作為本地的 AI 節點。


**細胞壁 (The Cell Wall)**：使用 Docker Container 確保進程隔離，單點故障不會影響整體伺服器 。

## 4. 感官、交互與情境分流

**神經網絡 (Telegram 整合)**：透過 Telegram API 作為控制接口，實現隨時隨地接入數位大腦 。

**聽覺感官 (Whisper 本地部署)**：本地安裝 Whisper 模型，傳送 Telegram 語音訊息時，AI 可自動轉錄並理解，無需額外 API 費用 。這與您之前在舊設備上成功運行 Whisper 節點的經驗完美契合。

**情境分流**：建立獨立的專屬對話群組（如：創業想法、會計事務、代碼開發），避免記憶污染 。

## 5. 技能與進化 (Skills & Evolution)

**技能解剖 (skill.md)**：技能由 Markdown 描述檔加上執行腳本（Python/Bash）組成 。

**編碼代理人 (GitHub 整合)**：為 Bot 註冊獨立的 GitHub 帳號 。允許其編寫代碼並 `commit` 到倉庫，實現自我進化 。



## 6. 記憶堆疊與自動化心跳

**記憶堆疊**：
**短期記憶 (Session Memory)**：暫存空間，約保存 2 天 。

**長期永久記憶**：啟用 **Qdrant 向量資料庫**，透過 LLM Embedding 提升語義搜尋與回憶能力 。

**數位心跳 (自動化與自我修復)**：透過 `cron job` 設定每 30 分鐘觸發一次喚醒 (Wake Up) 與執行維護（如 Email Check） 。

**子代理人 (多工處理)**：主代理人可生成多個子代理人（例如啟動 10 個子代理人）平行處理不同任務背景，最後彙整報告 。



## 7. 定義自我與下一步

**定義身份**：不要手動填寫資料。要求 Bot 對您進行「面試」 (The Interview Method)，由它來完善 `user.md`（關於你）與 `identity.md`（關於它）檔案，定義性格與核心價值 。

**啟動共生**：調整 `soul` 文件，安裝自定義技能。OpenClaw 的價值在於你如何教導它成為你的延伸 。
